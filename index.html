Data - Den endelige aflevering

Opgave

Bunden forudsætning

I denne afleveringsopgave skal du lave en web app, der henter og viser data fra en JSON-datakilde. I definerer i fællesskab JSON-data, som du skal hente via JavaScript. Med JavaScript skal du på baggrund af den hentede JSON-data, generere HTML-elementer, så du kan visualisere data fra JSON-filen. Både web app og JSON-data skal publiceres via GitHub og GitHub Pages. Undervisningen er tilrettelagt, så du gradvist lærer, hvordan du skal løse denne endelige aflevering – frygt derfor ikke, når du ser listen af krav herunder.

Krav

Du skal demonstrere, at du forstår og kan anvende teknologier, emner og koncepter fra data-forløbet. I dette afsnit er kravene for aflevering beskrevet, men se det gerne som en helhed på bagrund af den undervisning, du har været igennem i data-forløbet og på 1. semester.

Teknologier og programmeringskoncepter

  • HTML, CSS og JavaScript - du må ikke anvende libraries eller frameworks, heller ikke CSS stylesheets, som du ikke selv har skrevet. Du må gerne lade dig inspirere af fx csslayout.io, w3schools, eksisterende GitHub Repositories - men lad være med at kopiere direkte.
  • DOM-manipulation, herunder textContent, insertAdjacentHTML og/eller createElement
  • Fetch af data fra en online JSON-datakilde
  • Await og async
  • Variabler, objekter, arrays, og loops (foreach og/eller for of).
  • String-konkatenering
  • funktioner med parametre og returværdier
  • Events og closure
  • HTML-elementer som ul, ol, li, button, header, section, footer
  • CSS Grid, CSS Flexbox og/eller HTML Table
  • Separation of Concerns - din kode skal være opdelt i mindre specialiserede dele (funktioner) for at opnå god struktur.

User Interface

Din løsning skal bestå af følgende UI-elementer: Et grid eller en tabel: Du skal vise data fra din JSON-datakilde i et grid eller en tabel. Du skal kun vise udvalgte properties i grid eller en tabel. Et detail view: Der skal være et detail view, hvor du viser alle properties. Detail view kan implementeres med dialog-elementet. Derudover skal du have en header og en footer.
